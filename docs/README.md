## 📋 Checklist de Documentação

Ao trabalhar no projeto, use este checklist para garantir que a documentação está atualizada:

### Ao Criar um Novo Componente

- [ ] Componente tem bloco JSDoc completo no cabeçalho
- [ ] Componente está documentado em [COMPONENT_LIBRARY.md](./COMPONENT_LIBRARY.md)
- [ ] Exemplo de uso adicionado em [COMPONENT_USAGE_EXAMPLES.md](./COMPONENT_USAGE_EXAMPLES.md)
- [ ] Acessibilidade implementada (consulte [ACCESSIBILITY_GUIDE.md](./ACCESSIBILITY_GUIDE.md))

### Ao Criar uma Nova Tela

- [ ] Tela segue padrões de [DESIGN_PATTERNS.md](./DESIGN_PATTERNS.md)
- [ ] ViewModel criado seguindo [ARCHITECTURE.md](./ARCHITECTURE.md)
- [ ] Testes escritos e documentados
- [ ] CHANGELOG.md atualizado

### Ao Modificar Arquitetura

- [ ] [ARCHITECTURE.md](./ARCHITECTURE.md) atualizado
- [ ] [AI_CONTEXT.md](./AI_CONTEXT.md) atualizado se necessário
- [ ] [DESIGN_DECISIONS.md](./DESIGN_DECISIONS.md) atualizado com justificativa
- [ ] CHANGELOG.md atualizado

### Ao Encontrar um Bug

- [ ] Verificado em [KNOWN_ISSUES.md](./KNOWN_ISSUES.md) se já é conhecido
- [ ] Se novo, adicionado em [KNOWN_ISSUES.md](./KNOWN_ISSUES.md)
- [ ] Workaround documentado se aplicável

## 📝 Convenções de Documentação

### Formato

- Todos os documentos usam Markdown
- Títulos seguem hierarquia clara (H1 → H2 → H3)
- Código sempre com syntax highlighting
- Exemplos práticos sempre incluídos

### Estrutura

Cada documento técnico deve conter:

1. **Índice** - Navegação rápida
2. **Visão Geral** - Contexto e propósito
3. **Conteúdo Principal** - Detalhes técnicos
4. **Exemplos** - Casos de uso práticos
5. **Referências** - Links para documentação relacionada

### Atualização

- Documentação deve ser atualizada junto com o código
- CHANGELOG.md deve refletir mudanças na documentação
- Blocos JSDoc devem ser mantidos atualizados

## 🆘 Precisa de Ajuda?

### Não Encontrou o que Procura?

1. Verifique este índice completo
2. Use a busca do seu editor/IDE
3. Consulte [KNOWN_ISSUES.md](./KNOWN_ISSUES.md) para problemas comuns

### Encontrou um Erro na Documentação?

1. Verifique se o problema já está documentado em [KNOWN_ISSUES.md](./KNOWN_ISSUES.md)

3. Atualize o documento com a correção
4. Atualize o CHANGELOG.md
